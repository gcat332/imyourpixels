# Pixel Mood Booster — Design Spec

## Goal

สร้างเว็บหน้าเดียวชื่อ “แวะมายิ้มก่อน” สำหรับให้คนที่กำลังใจร้อนหรืออารมณ์ไม่ดีเข้ามาเล่นสั้นๆ แล้วรู้สึกผ่อนคลายและยิ้มได้ เว็บต้องเปิดจากลิงก์ GitHub Pages ได้โดยไม่พึ่ง backend หรือระบบล็อกอิน

## Experience

ผู้ใช้เห็นฉากด้านข้างแบบเกม pixel art: ถนนญี่ปุ่นยุค 90s ร้านขนมริมทาง ป้ายไฟ ตู้กดน้ำ และรถ JDM ขับเข้ามาจอดหน้าร้านโดยอัตโนมัติ จากนั้นผู้ใช้กด “จอดพักแป๊บ” เพื่อเริ่มกิจกรรมสั้นๆ ใช้เวลารวมประมาณ 30–60 วินาที

กิจกรรมหลักเป็นมินิเกมกด/แตะง่ายๆ เช่น กดรับขนมที่เด้งออกจากร้านหรือกดเก็บหัวใจ โดยไม่มีแพ้ ไม่มีตัวจับเวลา และไม่มีเสียงอัตโนมัติที่รบกวน ผู้ใช้กดเปิดเสียงได้เอง เมื่อจบจะแสดงข้อความอบอุ่นหรือกวนเล็กน้อย เช่น “ยิ้มได้แล้ว ค่อยไปต่อนะ” และรถขับออกจากฉากพร้อมเอฟเฟกต์หัวใจ/ดาว

## Visual direction

- ทุกองค์ประกอบเป็น pixel art: ฉาก รถ ตัวละคร ปุ่ม กล่องข้อความ และเอฟเฟกต์
- มุมมองด้านข้างแบบเกมยุค 90s
- สีหลัก: ชมพูพาสเทล ฟ้า ครีม เหลือง และแดงนีออน
- รถหลักเริ่มต้นเป็น Nissan Skyline GT-R R32 โทนฟ้าอมม่วง; โครงสร้างเผื่อเพิ่ม RX-7 FC/FD และ RX-5 ภายหลัง
- ร้านขนมเป็นจุดพักอารมณ์ ไม่ใช้ถ้อยคำตำหนิหรือบังคับให้ผู้ใช้ “ใจเย็น”
- ใช้ฟอนต์ pixel ที่อ่านง่ายและรองรับภาษาไทย หากไม่มีฟอนต์ภายนอกให้มี fallback ที่ชัดเจน

## Interaction and state

สถานะของฉาก: `arriving` → `parked` → `playing` → `finished` → `departing` โดยปุ่มหลักเปลี่ยนตามสถานะและผู้ใช้ต้องเริ่มเสียงเองผ่านปุ่มเล็กๆ

การกดซ้ำต้องไม่ทำให้สถานะค้างหรือมินิเกมเริ่มหลายครั้ง การโหลดหรือการเล่นโดยไม่มีเสียงต้องใช้งานได้เหมือนเดิม และต้องรองรับทั้งคลิกเมาส์และการแตะบนมือถือ

## Technical approach

- ใช้ HTML, CSS และ Vanilla JavaScript แบบ static site
- เก็บ asset แยกใน `assets/sprites`, `assets/backgrounds` และ `assets/sounds`
- ใช้ CSS animation เป็นหลักสำหรับรถ เมฆ แสง และเอฟเฟกต์
- ใช้ JavaScript จัดการ state, มินิเกม, ข้อความสุ่ม และ `prefers-reduced-motion`
- ไม่มี backend, analytics, login หรือข้อมูลส่วนตัว
- Deploy ด้วย GitHub Pages จาก branch ที่กำหนด หรือ GitHub Actions หากต้องการ build ภายหลัง

## Accessibility and responsive behavior

- มีปุ่มและข้อความที่ใช้ได้ด้วยคีย์บอร์ด
- ปุ่มหลักมี `aria-label` และสถานะประกาศด้วยข้อความที่อ่านได้
- ฉากปรับตามความกว้างหน้าจอ โดยคงอัตราส่วน pixel art และเพิ่ม hit area บนมือถือ
- ลด/หยุด animation เมื่อผู้ใช้เปิด `prefers-reduced-motion`
- ไม่ใช้สีอย่างเดียวเพื่อสื่อสถานะ

## Verification

- เปิดหน้าเว็บใน browser โดยตรงได้
- ทดสอบลำดับการเล่นตั้งแต่รถเข้า → กดเริ่ม → เล่น → จบ → รถออก
- ทดสอบคลิก คีย์บอร์ด และ touch
- ทดสอบหน้าจอมือถือและ desktop
- ตรวจสอบว่าไม่มี asset path ที่พังเมื่อรันใต้ GitHub Pages project path
